# Design: Data Fidelity Hardening

**Status:** active
**Date:** 2026-05-04
**Upstream:** Research audit (2026-05-04), `.rpi/reviews/full-attestation-audit-2026-05-03.md`

## Problem

Six data fidelity issues threaten production-grade Gloranthan correctness:

1. **loadCharacter() accepts invalid data** — blind `Object.assign()` lets placeholder skills, corrupted names, and invalid types enter the character via JSON import without validation.
2. **11 AiG folk magic spells missing from master list** — culture-specific spells from AiG (Find Truth, Preserve, Incognito, etc.) cannot be selected in career folk magic step because `FOLK_MAGIC_SPELLS` only contains Mythras Core spells.
3. **"Longbow or Javelin" unresolved** — Sartarite Hill Clan Levy has a literal "or" in the weapons array that renders verbatim instead of resolving to one weapon.
4. **OCR artifacts in CULTS_DATA** — newlines (`\n`) and unicode superscript markers (¹²⁶⁹) corrupt skill/spell names from Notes from Pavis extraction.
5. **No tech-level weapon filtering** — BACKGROUND_EQUIPMENT gives Greatswords to Balazarings (Primitive, stone/wood culture) because equipment generation ignores culture technology.
6. **Cult skills unresolved after cult selection** — `Devotion (Pantheon, Cult or God)` survives into final character data verbatim instead of resolving to the chosen cult name.

## Source Precedence

Per ADR-003:
1. AiG (GenCon 2015) — cultures, combat styles, folk magic, equipment
2. Mythras Core — careers, standard/professional skills, base spell list
3. Notes from Pavis — cult data (after OCR cleanup)
4. House rules (documented in ADRs)

## Design

### Fix 1: loadCharacter() validation

Add validation layer between JSON parse and `Object.assign()`:

```javascript
loadCharacter() {
  // ... file reading ...
  const data = JSON.parse(e.target.result);
  const warnings = validateLoadedData(data);
  if (warnings.blockers.length > 0) {
    this.showToast(`Cannot load: ${warnings.blockers[0]}`, 'error');
    return;
  }
  Object.assign(CharacterData, data);
  if (warnings.fixable.length > 0) {
    this.showToast(`Loaded with ${warnings.fixable.length} auto-corrected issues`, 'warning');
  }
  // ... rest of load ...
}

function validateLoadedData(data) {
  const warnings = { blockers: [], fixable: [] };
  // Check all skill maps for unresolved placeholders
  for (const skillMap of ['culturalSkills', 'careerSkills', 'bonusSkills']) {
    if (data[skillMap]) {
      for (const key of Object.keys(data[skillMap])) {
        if (needsDisambiguation(key)) {
          warnings.blockers.push(`Unresolved skill "${key}" — open in Wizard Mode to fix`);
        }
      }
    }
  }
  // Check selectedProfessionalSkills array
  if (data.selectedProfessionalSkills) {
    for (const skill of data.selectedProfessionalSkills) {
      if (needsDisambiguation(skill)) {
        warnings.blockers.push(`Unresolved skill "${skill}" — open in Wizard Mode to fix`);
      }
    }
  }
  return warnings;
}
```

**Decision:** Block load entirely if unresolved placeholders are found (user must fix in Wizard Mode). This is strict but prevents corruption.

### Fix 2: Add 11 AiG-specific folk magic spells to master list

Add these attested spells to `FOLK_MAGIC_SPELLS`:
- `Find Truth` (Balazaring, Sartarite)
- `Preserve` (Balazaring, Praxian, Esrolian)
- `Incognito` (Esrolian)
- `Find Safe Shelter` (Balazaring)
- `Find Herd Animal` (Praxian)
- `Find Prey` (Telmori)
- `Find Food` (Praxian)
- `Heat` (Telmori)
- `Find Water` (Praxian) — resolving "Find (Water, Food, Shelter)" into separate spells

**Source:** AiG p.26-41 (culture folk magic lists). These are Gloranthan-specific variants not in Mythras Core's generic spell list.

**Decision:** Add them alphabetically to the FOLK_MAGIC_SPELLS array. They become selectable in Step 9 (career folk magic) as well as Step 5 (cultural).

### Fix 3: Resolve "Longbow or Javelin"

Replace `"Longbow or Javelin"` in Sartarite Hill Clan Levy with both weapons as separate entries: `["Longbow", "Javelin"]`.

**Source:** AiG p.33 lists Hill Clan Levy with "longbow or javelin" — meaning the style trains with both, not that the character must choose one. The combat style trait "Throw Weapons" confirms javelins are part of the style.

**Decision:** Include both weapons in the array. At character generation, the character receives whichever weapon(s) are generated by the equipment system. The combat style itself supports both.

### Fix 4: Clean OCR artifacts in CULTS_DATA

Programmatic cleanup:
1. Replace all `\n` within skill/spell name strings with empty string (join split words)
2. Strip unicode superscript characters (¹²³⁴⁵⁶⁷⁸⁹⁰) — these are footnote markers from the source PDFs
3. Normalize `"Combat style with sword"` → `"Combat Style (Sword)"` (consistent formatting)
4. Normalize `"Runic Affinityion"` → `"Runic Affinity"` (typo)

**Decision:** Fix in-place in CULTS_DATA. One-time cleanup, no ongoing mechanism needed since we won't re-import OCR data without manual review (per ADR-003).

### Fix 5: Tech-level weapon filtering for BACKGROUND_EQUIPMENT

Add culture-aware weapon pools that respect CULTURE_ARMOUR material constraints:

```javascript
const CULTURE_WEAPON_POOLS = {
  'Primitive': {  // Balazaring, Telmori
    basic: ['Club', 'Hatchet', 'Knife', 'Shortspear', 'Sling', 'Javelin', 'Dagger'],
    higher: ['Battleaxe', 'Longspear', 'Shortbow', 'Longbow']
  },
  'Barbarian': {  // Sartarite, Old Tarsh
    basic: ['Hatchet', 'Club', 'Knife', 'Shortspear', 'Sling', 'Dagger', 'Javelin'],
    higher: ['Broadsword', 'Battleaxe', 'Longsword', 'Longspear', 'Longbow', 'War Sword']
  },
  'Civilised': {  // Esrolian, Lunar Heartland, Lunar Provincial, God Forgot
    basic: ['Dagger', 'Knife', 'Shortsword', 'Shortspear', 'Buckler'],
    higher: ['Broadsword', 'Longsword', 'Mace', 'War Sword', 'Scimitar', 'Halberd']
  },
  'Nomad': {  // Praxian, Grazelander
    basic: ['Hatchet', 'Knife', 'Shortspear', 'Javelin', 'Sling', 'Dagger'],
    higher: ['Lance', 'Longspear', 'Shortbow', 'Composite Bow', 'Scimitar']
  }
};
```

Map cultures to tech levels and use the appropriate pool in equipment generation instead of the current universal lists.

### Fix 6: Auto-resolve cult-bound skills after cult selection

When a cult is selected in Step 9, resolve:
- `Devotion (Pantheon, Cult or God)` → `Devotion (CultName)`
- `Binding (Cult, Totem or Tradition)` → `Binding (CultName)`
- `Invocation (Cult, School or Grimoire)` → `Invocation (CultName)`

**Decision:** Resolve at the point of cult selection. Remove from `KNOWN_CONCRETE_SPECIALIZATIONS` whitelist (they are NOT concrete — they're waiting for cult context). Add to random generation: use the randomly selected cult name.

## Trade-offs

| Decision | Pro | Con |
|----------|-----|-----|
| Block load on invalid data | Prevents corruption | User must re-open Wizard Mode for old saves |
| Both Longbow + Javelin in array | Matches AiG intent, combat style supports both | Minor deviation from "or" text |
| Strip footnote markers | Clean data | Lose traceability to source PDF footnotes |
| Tech-level weapon pools | Gloranthan authenticity | More code to maintain |
| Strict cult skill resolution | No placeholders in output | Less flexible for multi-cult characters (not applicable in current system) |

## Out of Scope

- Theism pipeline (broken end-to-end per attestation audit — separate initiative)
- Re-extracting OCR data from source PDFs (one-time fix is sufficient)
- Adding career-specific suggestion lists for placeholder skills (per ADR-005)
- Combat trait attestation (100% gap — separate initiative)
