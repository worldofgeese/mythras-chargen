1. Source hierarchy

AGENTS.md project rule; ADR-003

  1. Adventures in Glorantha for Folk Magic, Rune Magic, cultures, and combat styles.
  2. Mythras Core Rulebook, 3rd Printing 2018 for base rules, Animism, Sorcery, and Mysticism.
  3. Notes from Pavis cult one-pagers for cult definitions and cult magic.
  4. Monster Island and A Bird in the Hand for spirit templates where used.
  5. Hannu house rules for rune casting, devotional pool, recovery, and rank progression.

Conflict rule: when this page or an ADR names a Hannu house rule, that campaign rule overrides vanilla Mythras or AiG for this table. The source hierarchy tells you where the base material comes from; the ADR tells you when the campaign intentionally changes it.

2. Theist Rune Magic uses Rune Affinity, not Exhort

ADR-002; ADR-0007; AiG p.24 and p.69-75

At this table, theist miracles are cast by checking access, choosing the spell's rune, rolling the matching Rune Affinity, and spending from the Devotional Pool. Exhort remains in some source data only as a legacy Mythras term.

CategoryPlayer-facing rule

4. Animism and Sorcery adaptations

ADR-0006; ADR-0007; AiG p.60; .rpiv/guidance/magic-system.md

  • Animism: Binding is replaced by Spirit Rune affinity for this campaign adaptation. Trance remains the skill for spirit-world awareness/discorporation unless the GM calls for Spirit Rune in a specific scene. Starting Spirit Worshippers use CHA/2 bound spirit slots.
  • Sorcery: Invocation maps to the spell's Rune Affinity. Shaping maps to the Law Rune. Starting sorcerers choose three spells.
  • Mysticism: Supported by the codebase for future data, but no current supported one-pager requires it at chargen.

5. Data rule

ADR-003

Do not treat a spell, cult, spirit, or extracted book-data summary as canonical unless it has a source trail: PDF -> reference JSON -> inline app data -> UI. House rules follow a different trail: source artifact or GM ruling -> ADR -> implementation. When in doubt, ask for the source trail instead of guessing.
