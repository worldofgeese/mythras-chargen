Attestation chain

Player-facing extracted book and data mechanics should follow this chain:

Source PDF -> reference JSON -> inline constant -> UI/PDF output.

House rules follow this chain instead: source artifact or GM ruling -> ADR -> implementation -> player handout.

  • Cult miracle lists: NfP PDFs -> references/theism-miracles.json -> MIRACLES_DATA -> Step 9 and Play Mode.
  • Culture and cult access: AiG/NfP references -> references/culture-cult-map.json -> culture/cult picker.
  • Magic systems: ADR-0006 and source JSON -> cult type detection and system-specific pickers.
  • House rules: Hannu artifact/Discord guidance -> ADR-002 and ADR-0007 -> displayed rules for Rune Affinity, Devotional Pool, and recovery.

What to ask when uncertain

  • Is this an official Mythras rule, an AiG adaptation, NfP cult data, or a Hannu house rule?
  • Does this spell or spirit appear in a reference JSON?
  • Does this rule apply at chargen, advancement, or only during play?
  • Is this already implemented in the character generator or only documented for the table?
