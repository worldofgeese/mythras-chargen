## Part A: PDF Export Analysis & Solution

### Current State (Before)

The application had **three export/print paths**:

| Path | Method | Output |
|------|--------|--------|
| **Export PDF** | `App.exportPDF()` | Multi-page PDF (loaded user-provided template, filled form fields) |
| **Print Sheet** | `App.printSheet()` → `window.print()` | Browser print dialog (single-page via CSS @page rules) |
| **Export HTML** | `App.exportHyperclay()` | Self-contained HTML file with print button |

**Issues**:
- Export PDF required user to provide template file
- Template was multi-page (3 pages documented in PLAN.md)
- Inconsistent with single-page requirement

### Solution Implemented

**Replaced** `App.exportPDF()` with a **single-page generator** using pdf-lib:

**Changes made** (`index.html` lines 3429-3535):
1. Removed template file picker (`<input type="file">`)
2. Generate PDF from scratch using `PDFDocument.create()`
3. Single A4 page (595×842 points)
4. Compact layout with all essential data:
   - Character identity (name, culture, career, homeland, age)
   - 7 characteristics (STR, CON, SIZ, DEX, INT, POW, CHA)
   - 6 attributes (Action Points, Initiative, Damage Mod, Healing, Movement, Luck)
   - 7 hit locations with HP values
   - Up to 40 skills (two-column layout)
   - Weapons and equipment (up to 5)
   - Magic spells (up to 15)
   - Trademark footer (Mythras/TDM)

**Button tooltip updated** (line 648):
- Old: "Select your Mythras PDF template to fill with character data"
- New: "Generate a single-page PDF character sheet"

### PDF Export Paths After Implementation

| Path | Output | Single-Page? |
|------|--------|--------------|
| **Export PDF** | Generated single-page PDF | ✅ Yes |
| **Print Sheet** | Browser print (CSS @page A4 portrait) | ✅ Yes |
| **Export HTML** | Standalone HTML with print button | ✅ Yes (via print CSS) |

**All three paths now produce single-page output.**

### How to Generate Single-Page PDF

1. Complete character creation wizard (11 steps)
2. Switch to Play Mode
3. Click **"Export PDF"** button
4. PDF downloads as `{character-name}-sheet.pdf`
5. No template file needed — generated from CharacterData

### Layout Design

Structured to match typical RPG character sheet:
- **Top**: Title and character identity
- **Upper section**: Characteristics boxes (horizontal) + attributes grid
- **Middle section**: Hit locations + skills table (two columns)
- **Lower section**: Combat/equipment + magic spells
- **Footer**: Generator credit + Mythras trademark

Visual composition is **print-suitable** with proper margins (40pt) and readable font sizes (8-12pt).

## Part B: Decapod Rewrite Investigation

### What is Decapod?

**Decapod** is part of the OpenSpec/OpenClaw automated validation system for software projects.

**Evidence**:
- `CONTINUE.md` line 64: Listed as "Pending Task" — "OpenSpec + Decapod validation"
- Git commit author: kypris@openclaw.ai (OpenClaw infrastructure)
- No Decapod-specific config files in repo

### Root Cause of File Rewrites

**Decapod does NOT directly rewrite AGENTS.md or CLAUDE.md.**

The rewrites are caused by the **"eject" skill** documented in `PLAN.md` lines 127-140:

**Eject Skill Workflow**:
- Location: `~/.openclaw/workspace/skills/eject/SKILL.md`
- Trigger: User says "eject" or "eject this project"
- Actions:
  1. Gather all project files
  2. **Generate CLAUDE.md** (Claude Code project config)
  3. **Generate CONTINUE.md** (session handoff notes with timestamp)
  4. Create zip package

**Evidence from CONTINUE.md**:
```
Generated: 2026-04-03T13:45 CEST
Source: OpenClaw session 2026-04-02/03
```

**File roles**:
- `CLAUDE.md` — Project config for AI sessions (regenerated by eject)
- `AGENTS.md` — Source hierarchy rules (should be stable, in git)
- `CONTINUE.md` — Session artifact (intentionally regenerated each session)

### Prevention Strategies

| Strategy | How to Implement |
|----------|------------------|
| **1. Don't trigger eject** | Avoid saying "eject" unless you want project handoff package |
| **2. Git protection** | Commit files to version control; use `git checkout --` to restore |
| **3. Customize eject skill** | Edit `~/.openclaw/workspace/skills/eject/SKILL.md` to skip certain files |
| **4. Accept CONTINUE.md** | Let it regenerate (it's a session artifact); keep permanent context in AGENTS.md |

**Recommended workflow**:
1. Keep in version control: `AGENTS.md`, `CLAUDE.md`, `README.md`, `PLAN.md`
2. Let regenerate freely: `CONTINUE.md` (ephemeral session notes)
3. Review before committing: `git diff` after any eject operation
4. Restore if needed: `git checkout -- AGENTS.md CLAUDE.md`

### Configuration

**No Decapod-specific configuration files exist**:
- No `.decapod`
- No `.opsx`
- No `.openspec/` directory

Behavior is controlled by the eject skill implementation in OpenClaw workspace.
